Verification Methodology

Claims are tracked as data and rendered in UI through claim markers.

Data Model

  • sources.json: source registry and citation numbering.
  • claims.json: claim text, bindings, status, timestamps.
  • data-claim-id in HTML: binding between UI text and claim registry.

Status Semantics

  • verified: directly supported by source evidence.
  • partially_verified: partially supported, needs narrower wording.
  • conflicted: source contradicts current wording.
  • unverified: no stable source found yet.

Freshness Rules

Both last_verified_at (claim) and accessed_at (source) are required and validated in CI.
