Migration Strategy

Use a staged workflow to reduce risk when moving between coding agents.

Recommended Sequence

  1. Inventory current commands and extension dependencies (skills/slash/hook/MCP).
  2. Map command categories first, not tool names first.
  3. Migrate MCP and session continuity before advanced extension behaviors.
  4. Run verification checks and mark unresolved claims as unverified.

Common Failure Modes

  • Assuming all plugin systems are equivalent.
  • Ignoring non-interactive execution differences in CI scripts.
  • Treating partially verified claims as hard guarantees.
